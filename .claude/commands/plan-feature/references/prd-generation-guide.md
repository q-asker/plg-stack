# PRD 생성 가이드

1인 개발자/소규모 프로젝트를 위한 PRD(Product Requirements Document) 생성 가이드.
기업용 복잡성을 배제하고 바로 개발 가능한 실용적 명세만 작성한다.

## 시작 전 필수 작업

**반드시 `CLAUDE.md`를 읽어서 프로젝트의 기술 스택, 아키텍처, 모듈 구조를 파악한다.**
PRD의 기술 스택 섹션은 CLAUDE.md에 명시된 기술 스택을 기반으로 작성한다.

## 절대 생성하지 말 것 (IMPORTANT)

- 개발 우선순위
- 성능 지표
- 인프라
- 마일스톤
- 개발 단계
- 개발 워크플로우
- 보안 요구사항
- 페르소나

## 문서 정합성 보장 원칙 (CRITICAL)

**모든 섹션은 상호 참조되고 일관성을 유지해야 함:**

1. **기능 명세의 모든 기능**은 반드시 **API 엔드포인트**와 **모듈 매핑**에서 구현되어야 함
2. **모듈 매핑**에 있는 모든 기능은 **기능 명세**에 정의되어야 함
3. **누락 금지**: 한 섹션에만 존재하고 다른 섹션에 없는 기능은 절대 허용하지 않음
4. **중복 방지**: 같은 기능이 여러 모듈에 분산되지 않도록 명확히 구분

## 반드시 생성할 것 (IMPORTANT)

### 1. 프로젝트 핵심 (2줄)

- **목적**: 이 프로젝트가 해결하는 핵심 문제 (1줄)
- **타겟 사용자**: 구체적인 사용자층 (1줄)

### 2. 사용자 여정

- 전체 사용자 플로우 다이어그램 (API 호출 흐름)
- 주요 API 인터랙션 시퀀스
- 사용자 선택 분기점 명시

### 3. 기능 명세 (MVP 중심) — 정합성 기준점

- MVP에 반드시 필요한 핵심 기능만 포함
- **각 기능마다 기능 ID (F001, F002 등) 부여 필수**
- **각 기능이 구현될 모듈 명시 필수** (예: F001 → auth 모듈)

### 4. API 엔드포인트 구조

- RESTful API 엔드포인트 목록
- 각 엔드포인트별 기능 ID 매핑
- HTTP 메서드, 경로, 간략 설명

### 5. 모듈별 상세 기능

각 모듈마다:

- **역할**: 이 모듈의 핵심 목적
- **주요 기능**: 구체적 기능 목록
- **구현 기능 ID**: 이 모듈에서 구현되는 기능 ID 목록 **필수**

### 6. 데이터 모델

- 필요한 엔티티/테이블 이름 나열
- 각 테이블의 핵심 필드 (타입 포함)
- 엔티티 간 관계

### 7. 기술 스택

- **CLAUDE.md에 명시된 기술 스택을 그대로 반영**
- 새로 추가가 필요한 기술만 별도 제안

## 작성 가이드라인

1. **구체성**: "기능"이 아닌 "퀴즈 자동 채점 기능", "S3 이미지 업로드 기능"
2. **사용자 관점**: 기술적 구현이 아닌 사용자가 사용하는 기능 중심
3. **즉시 개발 가능**: 개발자가 이 문서만 보고 바로 코딩 시작할 수 있는 수준
4. **MVP 범위**: 프로젝트 성공에 반드시 필요한 최소 기능만 포함

## PRD 생성 프로세스 (정합성 보장)

1. **CLAUDE.md 읽기** — 기술 스택, 아키텍처, 모듈 구조 파악
2. 사용자 요청 분석
3. 전체 사용자 여정 플로우 설계
4. MVP 필수 기능만 추출 및 ID 부여
5. 각 기능별 구현 모듈 매핑
6. API 엔드포인트 구조 설계
7. 모듈별 상세 기능 명세
8. 데이터 모델 설계
9. 기술 스택 확인 (CLAUDE.md 기반)
10. **정합성 검증 체크리스트 실행**
11. 템플릿 형식으로 출력

## 정합성 검증 체크리스트 (PRD 완료 전 필수)

### 1단계: 기능 명세 → 모듈 연결 검증

- [ ] 기능 명세의 모든 기능 ID가 모듈별 상세 기능에 존재하는가?
- [ ] 기능 명세에서 명시한 관련 모듈이 실제 모듈별 상세 기능에 존재하는가?

### 2단계: API 엔드포인트 → 기능 연결 검증

- [ ] API 엔드포인트의 모든 기능 ID가 기능 명세에 정의되어 있는가?

### 3단계: 누락 및 고아 항목 검증

- [ ] 기능 명세에만 있고 모듈에서 구현되지 않은 기능이 있는가?
- [ ] 모듈에만 있고 기능 명세에 정의되지 않은 기능이 있는가?

**검증 실패 시: 해당 항목을 수정한 후 다시 전체 체크리스트 실행**
