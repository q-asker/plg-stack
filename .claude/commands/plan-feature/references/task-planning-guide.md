# Task 계획 가이드

Shrimp Task Manager로 태스크를 등록할 때 참조하는 가이드.

## 분석 방법론 (5단계)

### 1. 작업 계획

- PRD의 전체 scope와 핵심 기능들을 파악
- 기술적 복잡도와 의존성 관계 분석
- 논리적 개발 순서 및 우선순위 결정

### 2. 작업 분해

- 기능을 개발 가능한 Task 단위로 분해
- 각 Task는 독립적으로 완료 가능한 단위로 구성
- 1~2주 내 완료 가능한 범위로 분해

### 3. 구현 사항 명세

- 각 Task에 대한 구체적인 구현 사항 명시
- 체크리스트 형태의 세부 구현 내용 작성
- 수락 기준과 완료 조건 정의

### 4. 검증 기준 정의

- 빌드 성공, 린트/포맷 통과, 기능 동작 확인 등 구체적인 검증 항목 명시
- 자동화 가능한 검증(빌드, 테스트)과 수동 검증(UI 확인)을 구분
- UI/기능 검증 시 **Playwright MCP**를 활용하여 브라우저 기반 동작 확인

### 5. PRD 동기화

- Task 등록 완료 후 `docs/PRD.md`에 새 기능이 반영되었는지 확인
- PRD와 Shrimp Task 간 불일치가 없도록 교차 검증

## Task 작성 규칙

1. **독립성**: 다른 Task와 최소한의 의존성 유지
2. **구체성**: 추상적 표현보다 구체적인 기능 명시
3. **모듈 매핑**: 각 Task가 어느 모듈에서 구현되는지 명시
4. **검증 가능**: 완료 조건이 객관적으로 확인 가능해야 함

## 품질 체크리스트

### 기본 요구사항

- [ ] PRD의 모든 핵심 요구사항이 Task로 분해되었는가?
- [ ] Task들이 적절한 크기로 분해되었는가? (1~2주 내 완료 가능)
- [ ] 각 Task의 구현 사항이 구체적이고 실행 가능한가?

### 의존성 및 순서

- [ ] 기술적 의존성이 올바르게 고려되었는가?
- [ ] 중복 작업을 최소화하는 순서로 배치되었는가?

### PRD 동기화 검증

- [ ] 새로 추가된 기능이 PRD에 반영되었는가?
- [ ] PRD와 Shrimp Task 간 기능 목록에 불일치가 없는가?
